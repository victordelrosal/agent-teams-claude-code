Core Principles

5 Things Every AI Needs to Know

Before you dispatch your first agent team, internalize these.

01

Fresh context is the point

A subagent assigned to security review has zero knowledge of business context or architecture debates. That's good. It reads the code with genuinely fresh eyes. Independent perspective by design.

02

Specialization beats generalization

An agent given one job outperforms a general agent asked to "also handle" that job mid-task. Specialization in the system prompt shapes reasoning from the first token.

03

Decomposition is the hard skill

Good decomposition produces outputs that almost synthesize themselves. Poor decomposition produces confusion. The quality of your team depends almost entirely on how well you split the work.

04

Orchestrators coordinate, not execute

If your orchestrator is doing heavy analytical work, that's a design error. Orchestrators assign, collect, and synthesize. Subagents analyze, produce, and write. Keep roles separate.

05

The filesystem is your message bus

No orchestration framework needed. Agents write outputs to designated paths. Orchestrators read those paths. Simple, auditable, debuggable. This is the entire coordination infrastructure.

06

Sequential is sometimes correct

Security review can't start until code is written. The editor can't work until the first draft exists. Respect dependencies. A sequential pipeline of specialists still beats a solo generalist.

Coordination Patterns

5 Patterns. Every Team Fits One.

Full documentation with ASCII diagrams and failure modes in PATTERNS.md.

Pattern 1

Orchestrator-Worker

N identical workers handle N independent tasks. Orchestrator dispatches and collects. Best for homogeneous parallel work.

Research 5 markets ‚Üí 5 agents ‚Üí merge
Pattern 2

Pipeline

Sequential specialists where each stage feeds the next. Use when tasks have genuine dependencies.

Research ‚Üí Outline ‚Üí Write ‚Üí Edit
Pattern 3

Specialist Team

Different expertise on the same artifact or problem space. Orchestrator routes inputs and synthesizes.

Security + Perf + Style review same code
Pattern 4

Recursive/Nested

Agents spawn their own subagents for complex decomposition. Max depth 3. Use for very large tasks only.

Manager ‚Üí Team Leads ‚Üí Workers
Pattern 5

Review Loop

Worker produces, reviewer evaluates, worker refines. Iterative until quality threshold met. Max iterations guard required.

Writer ‚Üí Critic ‚Üí Writer ‚Üí Critic...
